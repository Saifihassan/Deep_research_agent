Overview:
- A small agent pipeline that: plans web searches, runs searches, composes a report, and can send email.
- Built on an `agents` runtime (openai-agents / custom Runner and Agent classes).

Top-level files and purpose:
- main.py : Orchestration / workflow entry point. Implements async functions that call agents via Runner.run:
  - plan_searches(query) -> calls Planner_agent and returns a WebSearchPlan
  - perform_Searches(searchplan) -> iterates searchplan.searchplan and calls search() for each term
  - search(item: WebSearchTerm) -> calls search_agent (uses searchtool) to perform web lookup
  - write_report(searchTerm, searchResults) -> calls writer_agent to produce reportSchema
  - emailsender(reports) -> calls emailer_agent which uses send_email tool
  - main() executes the pipeline (runs plan -> searches -> write -> email)

- ai_agents.py : Agent and tool definitions, model clients, and Pydantic schemas.
  - AsyncOpenAI clients: groq_client, gemini_client, ollama_client, openrouter_client (configured from env)
  - model1: OpenAIChatCompletionsModel configured with an AsyncOpenAI client
  - function tools:
    - searchtool(query): uses DuckDuckGo (DDGS) to fetch search text
    - send_email(subject, html_body): uses `resend` SDK to send email
  - Agents:
    - search_agent: uses searchtool to produce concise summaries per term
    - Planner_agent: generates a WebSearchPlan (output_type=WebSearchPlan)
    - writer_agent: generates a final report (output_type=reportSchema)
    - emailer_agent: composes and sends email using send_email tool
  - Pydantic schemas:
    - WebSearchTerm (reason, searchTerm)
    - WebSearchPlan (searchplan: list[WebSearchTerm])
    - reportSchema (summary, markdown_report, follow_up_questions)

- ui.py : (lightweight / placeholder) intended Gradio UI; may be minimal in current state.
- requirements.txt : Python dependency list (openai, openai-agents/agents, ddgs, pydantic, etc.)
- .env : environment for API keys (not checked in)
- README.md, .gitignore : project metadata

How the system flows (high level):
1. `main.main()` calls `plan_searches(query)` → Runner.run(Planner_agent, query) → returns WebSearchPlan.
2. `perform_Searches()` iterates the parsed `WebSearchPlan.searchplan`, calling Runner.run(search_agent, input) for each term.
3. The collected search results are passed to `write_report()` which calls Runner.run(writer_agent, input).
   - Because `writer_agent` has `output_type=reportSchema`, the agents runtime attempts to parse the model output into the Pydantic schema.
4. `emailsender()` calls Runner.run(emailer_agent, report.markdown_report) which triggers `send_email` tool to send mail.

Key implementation details and failure modes:
- The agents runtime validates model output into pydantic types when `output_type` is set. That requires the model to return raw JSON matching the schema exactly.
- A common error: the model returns JSON wrapped in Markdown code fences (```json { ... } ```). The runtime's JSON parser fails and raises a `ModelBehaviorError` / `ValidationError`.
- Recent mitigation added: a sanitizer fallback in `main.write_report()` that catches the parser error, extracts a JSON object from fenced code blocks or the first JSON-like substring, `json.loads` it, then validates via `reportSchema.model_validate(data)` and returns the parsed pydantic model if successful.

Run instructions:
1. Create and activate a Python virtualenv.
2. Install dependencies:
   pip install -r requirements.txt
3. Set environment variables (examples):
   - GROQ_API_KEY, GEMINI_API_KEY, OPENROUTER_API_KEY, RESEND_APIKEY
   - Optionally set debugging flags: OPENAI_DEBUG=true
4. Run the pipeline:
   python main.py
5. (Optional) Launch UI (if implemented):
   python ui.py
